Description:

SHORT TEXT
 Beispiel 0184: Überprüfungsdatum für Zahlungsversprechen setzen

FUNCTIONALITY
 With the function module processed for event 0184, you can define when
 the fulfillment of the promise to pay is to be checked and valuated.
 During the check, the level of fulfillment is determined. At the same
 time, a closing status can be assigned or a new check date defined.
 The following data is available:
 <LS>I_DFKKPP</>,,Header of the promise to pay.
 <LS>I_DFKKPPD_TAB</>,,Promised amounts and payment dates
 The system expects the following data to be returned:
 <LS>E_CHDAT</>,,Next check date
