SHORT TEXT
 INV: Create Additional (Customer-Specific/Industry-Specific) Documents

 trados = fi
FUNCTIONALITY
 With the function module called for event 2650, you can create own
 posting documents in invoicing that are to be integrated in the
 invoicing document. You can create the invoicing items for these posting
 documents yourself, or have them created automatically by the invoicing
 program.
 In this event you can use the data of the invoicing unit.
 The tables of posting document headers <LS>C_FKKKO_2650_TAB</>, business
 partner items <LS>C_FKKOP_2650_TAB</>, and general ledger items
 <LS>C_FKKOPK_2650_TAB</> can be transferred to the invoicing program. In
 table <ZK>C_INVDOC_I_2611_TAB</> you can transfer the prepared invoicing
 items.
 You can create one or several documents. You differentiate the documents
 using the temporary numbering of documents in field <LS>OPBEL</>.
 The invoicing program takes the posting documents unchanged and checks
 them for completeness and consistency.
 You can use a separate <DS:DE.CADOCTYPE_KK>reference document type</>
 (field <LS>FKKKO_2650-CADOCTYPE</>) per posting document.
 For each business partner item you can define whether the invoicing
 program is create an invoicing item automatically (by setting the
 indicator <LS>FKKOP_2650-</><ZK>INVDOC_I_REL</>).<ZK> </>You can also
 return the invoicing items you create in table <ZK>C_INVDOC_I_2611_TAB
 </>.
 For the purposes of identification and display in the invoicing
 document, you can assign the <DS:DE.INV_ITEMTYPE_KK>invoicing item type
 </> to the business partner items (in field <LS>FKKOP_2650-ITEMTYPE</>)
 and decide whether the amount of the business partner items is to be
 included in the final invoice amount (field <LS>FKKOP_2650-TOTALREL</>).
 The event is triggered in invoicing before the function modules for
 account maintenance are called. The postings documents created in this
 event are available for clearing, influence the final invoice amount,
 and can be displayed in the invoice.
EXAMPLE
 
HINTS
 If you create the invoicing items <ZK>C_INVDOC_I_2611_TAB </>yourself,
 make sure that for each posting document, the balance of the business
 partners with <ZK>TOTALREL </> (in the field <ZK>BETRW</>) agrees with
 the balance of the related invoicing items (also with <ZK> TOTALREL</>).
