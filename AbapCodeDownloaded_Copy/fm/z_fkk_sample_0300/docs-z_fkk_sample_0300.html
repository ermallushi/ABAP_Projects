SHORT TEXT
 INTERN FI-CA Mahnen: Mahnvorschlag nachbearbeiten

FUNCTIONALITY
 The function module called up at event 0300 enables you to postprocess
 the dunning proposal created in the dunning proposal run.
 Notes for Programming
 Internal table <ZH>I_FKKMAGRP</> contains the dunning group that later
 corresponds to the dunning header in the dunning proposal. Internal
 table <ZH>I_FKKMAVS</> contains the dunnable items which then form the
 dunning items in the dunning proposal.
 The reference is created via field <ZH>MGRPX</>, which is contained in
 both structures.
 The dunning groups that arise in this event do not necessarily form
 dunning notices. Dunning groups where either of the table fields
 <ZH>XLMST</> or <ZH>XMGRH</> is set are ignored for the creation of the
 dunning proposal.
 These fields can also be set in this event in order to integrate
 additional dunning notices.
 The dunning levels set by the dunning program are available in field
 <ZH>MAHNN</>. You can also change them there if necessary. If you do
 modify them, you should also modify the field <ZH>MSTNN</> (new dunning
 level type).
 The dunning levels of the documents before dunning are contained in
 field <ZH>MAHNS</> for information purposes.
 If you want to make changes to dunning notices in event 0300, you have
 to write corresponding messages in the application log of the mass
 activity, since a message about the creation of a duning notice may have
 already been written to the log (in event 0302).
